# UNIT-01（デザインシステム基盤）論理コンポーネント構成

`unit-01-nfr-design-plan.md`の回答（Q4, Q5）に基づく、UNIT-01のリポジトリ配置・トークン構造。

## リポジトリ配置（Q4=A）
デザインシステムは独立npmパッケージとしては切り出さず、フロントエンドアプリ内の1ディレクトリとして配置する。
- 配置例: `frontend/src/design-system/`
- 他機能ユニットのコード（`frontend/src/registration/`等）と同じビルド・パッケージに含まれる
- 各機能ユニットは`frontend/src/design-system/`配下のコンポーネントを直接importして利用する

## デザイントークン構造（Q5=B）
将来のテーマ切替（ダークモード等）を見据え、デザイントークンを最初からテーマ切替可能な構造で設計する。
- 色・タイポグラフィ・間隔等のトークンはCSS変数として定義し、テーマ別のCSS変数セット（例: `[data-theme="light"]` / `[data-theme="dark"]`）で切り替え可能な構造とする
- 初期リリースではライトテーマの値のみを定義する（ダークモードの実装自体は本ユニットのスコープ外。構造のみ将来の拡張に対応させる）

## 主要な論理コンポーネント（構造上の分類）
- **デザイントークン**: 色・タイポグラフィ・スペーシング・ブレークポイント（デスクトップ/タブレット2段階）のCSS変数定義
- **基本UIコンポーネント群**: ボタン・フォーム部品等（具体的なコンポーネント一覧・Props定義はCode Generation段階の計画で確定する）
- **ErrorBoundaryコンポーネント**: アプリケーション全体のトップレベルに1つ配置（DP-UNIT01-1）
- **i18n設定**: react-i18nextの初期化・プロバイダー
- **CSS Modules**: コンポーネント単位でスコープされたスタイル定義

## 具体的なコンポーネント一覧・Props設計について
本ユニットはFunctional Designをスキップしているため、個々のコンポーネント（Button, Input, Select等）の詳細なProps・振る舞いはこのNFR Designの対象外とする。Code Generation段階の計画（Part 1 - Planning）で具体化する。
