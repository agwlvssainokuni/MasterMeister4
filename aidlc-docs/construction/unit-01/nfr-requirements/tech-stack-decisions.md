# UNIT-01（デザインシステム基盤）技術スタック決定

`unit-01-nfr-requirements-plan.md`の回答（Q1〜Q3, Q9）に基づく、UNIT-01固有の技術選定。

## CSSスタイリング方式（Q1=A）
**CSS Modules**を採用する。
- コンポーネント単位でスコープされ、クラス名の衝突を防げる
- ビルドツール（Vite）が標準サポートしており追加設定が最小限
- ランタイムのCSS-in-JSライブラリを含まないため、NFR-6.1（サードパーティUIライブラリ不使用）の趣旨とも整合する

## 多言語対応（i18n）ライブラリ（Q2=A）
**react-i18next**を採用する。
- NFR-7.3（最初の実装ユニットからi18n基盤を導入）に対応
- デファクトスタンダードであり、名前空間分割・遅延ロード等、以降のユニットでの利用実績が豊富
- 初期リリース対応言語は日本語＋英語（NFR-7.1）。文言辞書は機能ユニットごとに名前空間を分割する方針とし、詳細は各ユニットのCode Generation段階で決定する

## アイコン方針（Q3=B）
このユニットでは最小限（共通コンポーネントの構築に直接必要なもの数個程度）のみを自作SVG Reactコンポーネントとして用意し、以降のユニットで必要になったアイコンをその都度追加する。
- 理由: 全アイコンの先行整備は本ユニットのスコープ（共通デザイン基盤の構築）を超え、Question 5個別画面デザインの段階的整備の方針（STORY-0.2、必要になったタイミングで整備する）とも整合する

## 依存パッケージ脆弱性スキャン（Q9=C、SECURITY-10）
- ローカルでの`npm audit`実行手順を整備し、Build and Testステージのテスト手順書に組み込む
- GitHub Dependabotのアラート設定は、UNIT-10（CI/CD）でリポジトリ設定として行う
- 依存パッケージは`package-lock.json`で厳密にバージョン固定する

## 既に確定済みの前提（requirements.mdより、再質問なし）
- サードパーティUIコンポーネントライブラリは使用しない（NFR-6.1）
- テストフレームワークはVitest + React Testing Library（NFR-9.1）
- レスポンシブはデスクトップ中心＋タブレット程度（NFR-8.1）
