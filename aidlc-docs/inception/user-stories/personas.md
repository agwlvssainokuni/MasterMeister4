# ペルソナ定義

`story-generation-plan.md` Q1=A（2ペルソナで十分。権限の違いはペルソナではなく個別のアクセス権限設定として扱う）に基づき、以下の2ペルソナを定義する。

---

## PERSONA-USER: 一般ユーザ

- **役割**: マスタデータの参照・編集・クエリ実行を行う業務担当者
- **主な目的**: 自分の業務に必要なマスタデータを、権限の範囲内で効率よく参照・修正・分析すること
- **典型的な操作**: マスタメンテナンス画面でのレコード参照・編集、クエリビルダーによるデータ抽出、保存クエリの実行、クエリ履歴の確認
- **権限との関係**: 個々のユーザまたは所属グループに設定されたアクセス権限（§5.2）の範囲でのみ操作可能。ペルソナとしては単一だが、実際の操作可能範囲は権限設定によって個人ごとに異なる
- **利用環境**: 社内デスクトップ環境（NFR-8.1に準拠、タブレット等でも大きく崩れない程度の対応）
- **モチベーション**: 手作業でのSQLクライアント操作や依頼ベースのデータ修正を減らし、自分自身で安全にマスタデータをメンテナンスしたい

## PERSONA-ADMIN: 管理者

- **役割**: システム全体の運用管理者。ユーザ・RDBMS接続・アクセス権限・監査ログを管理する
- **主な目的**: 安全にシステムを運用し、適切なユーザにのみ適切な権限を付与すること
- **典型的な操作**: ユーザ登録の承認／却下、対象RDBMS接続の登録とスキーマ取込、ユーザ／グループ単位のアクセス権限設定（個別設定・YAML入出力）、監査ログの確認
- **権限との関係**: 一般ユーザの操作範囲に加えて、管理者ダッシュボードへのアクセスと管理操作が可能。初期管理者アカウントは環境変数によるブートストラップで自動作成される（FR-1.14）
- **利用環境**: 一般ユーザと同様、社内デスクトップ環境
- **モチベーション**: 属人化しがちなデータベースアクセス権限の管理を、UIを通じて可視化・一元管理し、監査可能な状態を保ちたい

| 4. マスタメンテナンス | 一般ユーザ |

| 6. クエリ保存 | 一般ユーザ |
| 7. クエリ実行 | 一般ユーザ |
| 8. クエリ履歴 | 一般ユーザ・管理者 |
| 9. 監査ログ | 管理者（閲覧者） |
